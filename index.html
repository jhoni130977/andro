Menghitung Subnetting IP Class C

Cara / Rumus Subnetting IP Class C,Ada Softwarenya, Subnetmask, Maksud dari IP, Network Address, Broadcast Address, Subnet Mask. Bagi anda seorang anak TKJ (seperti saya) pasti sudah tidak asing lagi dengan kata Subnetting, lantas apa itu subnetting ?, berikut pembahsanya :

A. Apa Itu Subnetting
Subnetting adalah sebuah metode perhitungan sebuah IP jaringan, berdasarkan Class dari IP yang ingin di subnet dan subnet mask yang di gunakan. dengan menggunakan metode Subnetting ini kita bisa menentukan Network Address, Range IP yang bisa di gunakan, dan Broadcast Address.

berikut manfaat yang bisa kita dapatkan dari sebuah subnetting :
  • Memaksimalkan penggunaan IP yang akan di gunakan,
  • Kita bisa menentukan IP yang akan di gunakan sesuai kebutuhan, sehingga meminimalisir IP yang tidak terpakai.
  • Meningkatkan keamanan jaringan

B.1. Internet Protocol (IP)
Untuk istilah yang satu ini, mungkin sudah tidak asing lagi bagi anda, namun tak ada salahnya saya jelaskan kembali, IP singkatan dari Internet Protokol yang berfungsi sebagai alamat dari sebuah komputer yang berbentuk sederatan angka,yang di bagi menjadi empat bagian (oktet) dan di setiap bagianya mempunyai nilai (angka) dari 0 sampai 255, misal 192.168.1.1, tujuanya agar komputer tersebut bisa berkomunikasi dengan komputer lainya, seperti halnya transfer data dan berselancar di internet.

B.2. Network Address (NA)
Network Address merupakan IP pertama dalam sebuah jaringan dan tidak bisa di gunakan, NA bisa di katakan sebagai perwakilan dari IP yang akan di gunakan ketika berkomunikasi dengan komputer lain.

B.3. Range IP
Range IP adalah interval IP yang bisa di gunakan dalam sebuah jaringan, fungsi dari Range IP adalah kita bisa menentukan IP mana yang akan kita gunakan dan dapat memenuhi kebutuahn jaringan.

B.4. Broadcast Address (BA)
Sama halnya dengan NA, IP Broadcast address tidak bisa di gunakan dan merupakan alamat ip terakhir dari sebuah jaringan, funsinya yaitu untuk mengirimkan sebuah paket data ke semua user / host yang tergabung di dalamnya.

C. Menghitung Subnetting Class C
Untuk melakukan subnetting menurut saya sangatlah mudah, karena saya sudah tau rumus untuk subnetting, dan di artikel ini saya akan menulis ulang dan menjelaskan rumus subnetting yang telah saya buat sendiri, agar lebih mudah berikut saya contohkan :

C.1. Contoh Masalah
Seorang admin jaringan akan membuat sebuah jaringan LAN untuk 10 komputer / host, dengan IP 192.168.1.0, maka tentukanlah subnetmask, Nework address, Range IP yang di gunakan, dan BA.

C.2. Menentukan Subnetmask
Karena ini merupakan jaringan skala kecil (LAN), maka kita akan menggunakan IP Class C, dengan CIDR mulai dari 24 sampai 32, dan karena di soal hanya untuk 10 komputer, maka kita akan menggunakan CIDR /28
Mengapa 28 ?
karena CIDR 28 memiliki jumlah host yang bisa di gunakan dan mendekati kebutuhan 10 komputer yaitu 14 host (Lihat Tabel Subnet Mask - Poin B.1). selain dengan melihat tabel di atas, anda juga bisa mengetahui jumlah host dan network tanpa harus melihat / mengingat tabel. berikut pembahasannya :
Host = (2(32-CIDR)) - 2
sehingga :
Host = (2(32-28)) - 2
Host = (2(4)) - 2
Host = 16 - 2
Host = 14
sedangkan untuk menentukan jumlah network :

Network = 2(CIDR-24)
sehingga dapat kita ketahui jumlah network dari CIDR 28 adalah :

Network = 2(28-24)
Network = 2(4)
Network = 16

Mengapa tidak menggunakan CIDR lain, seperti 29 atau 27 ?
  • Karena jika menggunakan CIDR 29, maka akan kurang karena CIDR 29 hanya memiliki 6 host yang bisa di gunakan, sedangkan kebutuhan 10 host
  • Sedangkan subnet 27 akan kelebihan karena CIDR 27 memiliki memiliki 62 host yang bisa di gunakan, jika kita menggunakan 27, maka akan ada 52 host yang tidak terpakai dan rawan tindak kejahatan.
  • Kesimpulanya semakin besar nilai subnet maka jumlah host yang bisa di gunakan semakin sedikit,
  • Semakin kecil nilai subnet maka jumlah host yang bisa digunakan semakin banyak
Karena CIDR 28 yang di pakai, maka subnetmasknya adalah 255.255.255.240, 

C.3. Menentukan Network Address
Sebelum ke pembahasan selanjutnya, yaitu menentukan Network Address, pastikan anda sudah mengetahui jumlah Host dari subnet yang akan di pakai, karena rumus untuk mencara NA, ada kaitanya dengan Jumlah Host dan Jumlah Nework, berikut rumus dan contohnya :
NAn = (Host + 2) x (N - 1)
NA4 = (14 + 2) x (4 - 1)
NA4 = 16 x 3
NA4 = 48
Keterangan :
Host = Jumlah Host dari CIDR yang di gunakan
n = dari network ke berapa, NA mau di ketahui
n mengacu pada jumlah Network, misal untuk CIDR 28 mempunyai 1 - 16 network yang bisa di pilih.
NA4 = (14 + 2) x (4 - 1)
NA4 = 16 x 3
NA4 = 48

dari contoh di atas saya sedang mencari NA yang ke 4, dan hasilnya 48, sehingga dapat kita tuliskan Network address dari IP 192.168.1.0/28 memiliki NA 192.168.1.48 untuk network yang ke 4.
